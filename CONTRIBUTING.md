# Contributing

Thank you for contributing.

## Before opening a pull request

1. Open or reference an issue describing the problem.
2. Keep the change narrowly scoped.
3. Add or update tests where behavior changes.
4. Update documentation where user-facing behavior changes.
5. Do not include private, client, employer, regulated, or proprietary information.
6. Do not claim performance, compliance, readiness, or verification without evidence.

## Pull-request standard

A pull request should include:

- Problem
- Proposed change
- Verification performed
- Evidence or screenshots where relevant
- Risks and limitations
- Follow-up work, if any

By contributing, you agree that your contribution may be distributed under the repository's stated license.
